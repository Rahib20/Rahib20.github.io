Project 1

Roadsigns Multiple-Choice Quiz

A Visual Basic desktop application that quizzes users on UK road signs. It includes login and account creation, learn and test modes, and stores results in an Access database using SQL queries.

VB .NET Microsoft Access Relational database SQL queries
Screenshots  ·  walkthrough of key screens
1 / 10

Login screen: users enter their details and receive feedback if the login is successful or fails (e.g. “login failed” for invalid credentials).

Create account: the program validates the username and password. If any fields are empty, an error message prompts the user to complete the form.

Main menu: after a successful login, the main menu appears, giving the user three clear options to navigate around the program.

Category selection: when the user chooses test or learn mode, they are taken to the category screen, where they can pick from five different road sign categories.

Learn mode: displays a sign image, its name, and a description. Navigation buttons let users go through each sign in the selected category at their own pace.

Test mode: shows a sign and a multiple-choice question with four possible answers. The user selects their choice, and the program tracks their score.

Post-test summary: after a set of questions, the user can choose to review only their incorrect answers or browse all results.

Review incorrect answers: the application shows the sign, the question, and the user’s answer so they can clearly see what they got wrong and learn from it.

Results overview: results are stored and displayed in a table, including percentages for each category, giving a clear picture of performance.

Database design: all sign data is stored in a Microsoft Access database, including sign names, descriptions, image file paths, and category IDs – showing the relational structure behind the app.

This project helped me build confidence in form validation, user feedback, and working with relational databases. I used SQL queries from Visual Basic to insert and update records in the Access database, which is a skill I’ve carried into more recent projects.
